# QT-AI 화면정의서 v1.1

> 목적: 요구사항명세서, ERD, 현재 클릭 목업을 기준으로 화면별 정상/빈/오류/권한/데이터 상태와 이동 흐름을 빠짐없이 정의한다.  
> 기준 문서: `01.요구사항명세서_현재완성v1.md`, `02_ERD_문서_v2.md`, `03_화면설계_스토리보드.md`, `03_화면설계_클릭목업.html`

## 1. 설계 원칙

| 원칙 | 화면 반영 기준 |
|---|---|
| 로그인 강제 | 로그인 전에는 기능 화면을 제공하지 않는다. 토큰 만료 시 로그인 화면으로 보낸다. |
| 오늘의 QT 중심 | 로그인 후 홈 화면 없이 오늘의 QT로 진입한다. |
| 사용자 선택형 학습 | 요약, 해설, 용어, 시뮬레이터, TTS, AI 질문은 사용자가 선택할 때만 확장한다. |
| 기본 비공개 | 모든 노트는 기본 비공개다. 공유는 명시 동작과 닉네임 노출 안내 후 진행한다. |
| 자동저장 없음 | 노트 작성 화면은 `임시저장`과 `저장`을 분명히 구분한다. 자동 저장 상태를 만들지 않는다. |
| AI 검증 우선 | 검증 통과 콘텐츠만 노출한다. 실패·누락·검증중 콘텐츠는 준비중 또는 제한 안내로 처리한다. |
| 공유본 스냅샷 | 공유글은 공유 시점의 본문/닉네임/구절 스냅샷을 보여준다. 원본 수정과 자동 동기화하지 않는다. |

## 2. 공통 화면 상태

| 상태 | 적용 화면 | UX 처리 |
|---|---|---|
| 로딩 | 데이터 조회 화면 전체 | 본문 영역에 스켈레톤 또는 진행 표시. 하단 내비게이션은 유지 가능. |
| 빈 상태 | 목록, 알림, 찬양, 노트, 나눔 | 빈 이유와 다음 행동 1개를 제공한다. |
| 부분 실패 | QT 본문+해설, 마이페이지 위젯 | 핵심 본문은 유지하고 실패 영역에만 재시도 안내를 둔다. |
| 전체 실패 | 로그인, 본문 조회, 저장 | 실패 안내와 재시도 버튼을 제공한다. |
| 오프라인 | QT, 성경, 노트, AI, 알림 | 캐시 본문이 있으면 본문만 표시. 해설·AI·동기화는 사용 불가로 표시. |
| 권한 없음 | 관리자, 비공개 노트, 숨김 게시글 | 접근 거부 또는 볼 수 없는 콘텐츠 안내. 상세 데이터는 노출하지 않는다. |
| 토큰 만료 | 로그인 후 모든 화면 | 현재 화면 진입을 중단하고 로그인 화면으로 이동한다. |
| 저장 성공 | 작성·수정 화면 | 토스트 또는 완료 화면. 작성 내용과 저장 시각을 명확히 표시한다. |
| 저장 실패 | 작성·수정 화면 | 입력 내용은 화면에 유지하고 재시도 동선을 제공한다. |
| 삭제/숨김 대상 접근 | 나눔, 알림 링크, 댓글 | “삭제되었거나 볼 수 없는 글입니다” 안내를 표시한다. |

## 3. 데이터 상태와 화면 노출

| ERD 상태 | 화면 노출 기준 |
|---|---|
| `members.ACTIVE` | 일반 기능 사용 가능 |
| `members.SUSPENDED` | 로그인 가능 여부는 정책 확정 필요. 나눔 작성·댓글·신고 등 참여 기능은 제한 화면 필요 |
| `members.WITHDRAWN` | 로그인 불가 또는 탈퇴 완료 안내 |
| `qt_passages.DRAFT` | 일반 사용자에게 미노출. 관리자만 수정 가능 |
| `qt_passages.PUBLISHED` | 오늘의 QT에 노출 |
| `qt_passages.HIDDEN` | 일반 사용자에게 “오늘 QT가 준비되지 않았습니다” |
| `notes.DRAFT` | 작성자만 임시저장 목록/작성 화면에서 접근 |
| `notes.SAVED` | 작성자 노트 목록, 마이페이지 달력, 공유 가능 대상 |
| `notes.DELETED` | 작성자에게도 목록 미노출. 필요 시 삭제 완료 안내만 표시 |
| `notes.visibility.PRIVATE` | 작성자만 열람 |
| `notes.visibility.SHARED` | 작성자 노트와 연결된 공유글 진입 가능 |
| `sharing_posts.PUBLISHED` | 나눔 피드와 상세 노출 |
| `sharing_posts.HIDDEN` | 일반 사용자 미노출. 관리자 처리 상태 |
| `sharing_posts.DELETED` | 접근 시 삭제 안내 |
| `comments.PUBLISHED` | 댓글 목록 노출 |
| `comments.HIDDEN` | 일반 사용자 미노출 또는 운영 숨김 안내 |
| `comments.DELETED` | 댓글 목록에서 삭제 안내 또는 미노출 |
| `reports.RECEIVED` | 신고 접수 완료, 관리자 목록 노출 |
| `reports.REVIEWING` | 관리자 검토 중, 신고자 알림 가능 |
| `reports.RESOLVED` | 처리 완료 알림 생성 |
| `reports.REJECTED` | 반려 알림 생성 가능 |
| `ai_generated_assets.APPROVED` | 사용자 노출 가능 |
| `ai_generated_assets.VALIDATING/REJECTED/HIDDEN` | 사용자에게 원문 미노출 |

## 6. 오늘의 QT 화면 상세

### Q-01. 오늘의 QT 기본

| 구분 | 정의 |
|---|---|
| 화면 목적 | 오늘의 본문을 한/영 병렬로 읽게 하는 메인 화면 |
| 주요 데이터 | `qt_passages.PUBLISHED`, `qt_passage_verses`, `bible_verses` |
| 구성 | 날짜, 본문 제목/범위, 한글 본문, 영어 본문, 해설 펼치기, 시뮬레이터, TTS, AI 버튼, QT 노트 작성 |
| 기본 상태 | 해설/용어/요약 접힘, AI 버튼 우측 하단, 노트 작성 CTA 노출 |
| 빈 상태 | 오늘 QT 미등록 또는 숨김: “오늘의 QT가 준비되지 않았습니다” |
| 부분 실패 | 본문은 표시되나 해설·용어 실패 시 해당 영역만 재시도 |
| 오프라인 | 캐시 본문만 표시, 해설·AI·TTS 동기화 기능은 사용 불가 안내 |
| 이동 | Q-02, Q-03, Q-04, Q-05, Q-06 |

### Q-02. 절별 요약/해설/용어 펼침

| 구분 | 정의 |
|---|---|
| 화면 목적 | 사용자가 요청한 경우에만 본문 이해 보조 콘텐츠를 보여준다. |
| 노출 가능 | `verse_explanations.APPROVED`, `glossary_terms.APPROVED`, 출처 있음 |
| 미노출 | PENDING, REJECTED, HIDDEN, 출처 누락 |
| 상태 | 절별 승인됨, 한 줄 요약 없음, 해설 없음, 용어 없음, 출처 누락, 조회 실패 |
| 문구 | 준비된 해설 없음 / 해설을 불러오지 못했습니다 |
| 정책 | 실시간 AI 해설 생성 버튼을 두지 않는다. |

### Q-03. 시뮬레이터 보기

| 구분 | 정의 |
|---|---|
| 화면 목적 | 오늘의 QT 본문 장면을 검증된 장면 스크립트 기반으로 보여준다. |
| 적용 범위 | 오늘의 QT만 적용. 일반 성경과 자유 노트에는 미적용 |
| 노출 가능 | `simulator_clips.APPROVED` |
| 상태 | 재생 가능, 준비되지 않음, 검증중, 재생 실패 |
| 예외 | 클립 없음: 준비되지 않음 / 재생 실패: 재시도 |
| 정책 | 검증되지 않은 외부 이미지나 자유 형식 코드는 노출하지 않는다. |

### Q-04. AI 질문 슬라이드

| SYSTEM_BATCH | 화면 로그인 없이 사전 생성/집계 배치 실행 |

### AD-01. 관리자 대시보드

| 구분 | 정의 |
|---|---|
| 화면 목적 | 운영자가 오늘 처리할 콘텐츠·신고·검증 상태를 한눈에 확인한다. |
| 권한 | `admin_users.admin_role`에 따라 메뉴와 처리 버튼을 분리 |
| 예외 | 일반 사용자 접근 시 접근 거부 |

### AD-02. 오늘 QT 관리

| 구분 | 정의 |
|---|---|
| 화면 목적 | 날짜별 QT 본문 범위를 등록·수정·게시·숨김 처리한다. |
| 입력 | 날짜, 제목, 성경 권/장/시작 절/끝 절, 상태 |
| 검증 | 날짜 중복, 절 범위 오류, 시작 절 > 끝 절, 본문 없음 |
| 상태 | DRAFT, PUBLISHED, HIDDEN |
| 감사 로그 | 등록, 수정, 삭제/숨김, 게시 |

### AD-03. AI 산출물 검증

| 구분 | 정의 |
|---|---|
| 화면 목적 | 사용자 노출 해설·시뮬레이터·Q&A 응답 품질을 관리한다. |
| 대상 | EXPLANATION, SUMMARY, GLOSSARY, SIMULATOR, QA_RESPONSE |
| 상태 | VALIDATING, APPROVED, REJECTED, HIDDEN |
| 필수 표시 | 생성 지시 버전, 검증 체크리스트 버전, 실패 사유, 출처 표시 여부, 재생성 이력 |
| 액션 | 승인, 반려, 숨김, AI 재생성 트리거, 평가 셋 후보 등록 |
| 금지 | 검증용 한국어 주석 원문을 일반 관리자에게 노출하지 않음 |
| 예외 | 체크리스트 버전 누락 시 승인 불가 |

### AD-04. 신고 처리

| 구분 | 정의 |
|---|---|
| 화면 목적 | 신고된 게시글·댓글·AI 응답을 검토하고 처리한다. |
| 처리 | 유지, 비공개, 삭제, 사용자 제재, 반려 |
| 상태 | RECEIVED, REVIEWING, RESOLVED, REJECTED |
| 완료 | 신고자 알림 생성, 감사 로그 기록 |
| 예외 | 처리 실패 시 상태 변경 없음 |

### AD-05. 찬양 큐레이션 관리

| 구분 | 정의 |
|---|---|
| 화면 목적 | 운영자가 곡 메타데이터와 외부 링크를 관리한다. |
| 입력 | 곡명, 아티스트, 외부 링크 식별자, 상태 |
| 금지 | 가사 저장, 음원 파일 업로드, 사용자 디바이스 음원 업로드 |

### AD-06. 시스템 공지 관리

| 구분 | 정의 |
|---|---|
| 화면 목적 | 관리자가 사용자에게 노출할 시스템 공지를 작성·발행하고 인앱 알림 생성을 관리한다. |
| 입력 | 제목, 본문, 게시 상태, 게시 시작 시각 |
| 상태 | DRAFT, PUBLISHED, HIDDEN, 발행 실패, 알림 생성 실패 |
| 액션 | 임시저장, 발행, 숨김, 수정, 알림 발송 대상 확인 |
| 연동 | 발행된 공지는 `notifications.type=NOTICE`로 사용자 알림 목록에 노출된다. |
| 감사 로그 | 공지 작성, 수정, 발행, 숨김 |

### AD-07. 감사 로그

| 구분 | 정의 |
|---|---|
| 화면 목적 | 관리자와 시스템 배치의 주요 운영 변경 이력을 삭제 불가능한 추가 기록 방식으로 확인한다. |
| 대상 | QT 변경, 해설 상태 변경, AI 재생성 트리거, 신고 처리, 찬양 큐레이션 변경, 공지 발행, 사용자 제재, 검증 체크리스트 버전 변경, 컴포넌트 라이브러리 버전 변경 |
| AI 로그 대상 | AI Q&A 차단 사례, 자동 검증 실패, 평가 셋 반영, 반려 사유 변경 |
| 필터 | 작업자, 작업 유형, 대상 타입, 기간, 결과 |
| 상태 | 목록 있음, 검색 결과 없음, 조회 실패 |
| 정책 | 감사 로그는 수정·삭제 액션을 제공하지 않는다. |

### AD-08. AI 운영 모니터링

| 구분 | 정의 |
|---|---|
| 화면 목적 | AI 콘텐츠 생성·검증 품질과 운영 리스크를 지표로 확인한다. |
| 지표 | 검증 실패율, 반려 사유 분포, 재생성 횟수, 차단된 Q&A 유형, 검증 대기 건수, 체크리스트 버전별 통과율 |
| 액션 | 실패 건 상세 보기, 산출물 검증 화면 이동, 평가 셋 후보 이동, 재생성 대상 확인 |
| 상태 | 정상, 집계 지연, 집계 실패, 데이터 없음 |
| 정책 | 검증용 한국어 주석 원문과 비공개 원자료는 노출하지 않는다. |

## 12. 화면별 카피 원칙

| 상황 | 카피 기준 |
|---|---|
| 빈 상태 | 사용자가 잘못한 느낌을 주지 않고 다음 행동을 제안한다. |
| 저장 실패 | “저장되지 않았음”과 재시도 방법을 명확히 말한다. |
| AI 차단 | 기능 범위를 설명하고 판단·선택 대행은 하지 않는다고 안내한다. |
| 공유 안내 | 닉네임이 공개된다는 사실을 저장 전 확인시킨다. |
| 삭제/숨김 | 상세 이유를 과도하게 노출하지 않고 볼 수 없음을 안내한다. |
| 관리자 오류 | 상태 변경이 적용되지 않았음을 명확히 표시한다. |
