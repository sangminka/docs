# QT-AI 문서 정합성 관리표 v1.0

> **작성일:** 2026-05-17  
> **작성 관점:** 요구사항 의미 고정, 문서 간 추적성 관리, 의미 변경 후보 분리  
> **기준 문서:** `01.요구사항명세서_현재완성v1.md`

---

## 1. 관리 원칙

본 문서는 요구사항을 새로 정의하지 않는다. 이미 작성된 요구사항의 의미를 바꾸지 않고, 문서 간 기준선과 추적 관계를 고정하기 위한 정합성 관리 문서이다.

| 구분 | 처리 기준 | 승인 필요 |
|---|---|---|
| 정리 변경 | 버전명, 용어, 파일 참조, 문서 위치, 추적표, 중복 표현 정리 | 보통 불필요 |
| 의미 변경 | 기능 추가/삭제, 우선순위 변경, 정책 변경, 수용 기준 변경, 권한 변경 | 반드시 필요 |

정리 변경은 본문 요구사항의 의미가 유지되는 범위에서만 반영한다. 의미 변경 가능성이 있는 내용은 이 문서의 "의미 변경 후보"에 분리하고, 승인 전까지 기준 문서에 반영하지 않는다.

## 5. 의미 변경 후보

아래 항목은 요구사항 의미 또는 정책 확정에 영향을 줄 수 있으므로 승인 전까지 기준 문서에 반영하지 않는다.

| 후보 | 영향 영역 | 승인 필요 이유 |
|---|---|---|
| 회원 탈퇴 후 법적 보존 기간의 정확 일수 확정 | 요구사항, ERD, API, 개인정보 정책 | 보존 기간은 정책 변경에 해당 |
| AI Q&A 로그 저장 여부, 보존 기간, 학습 사용 여부 확정 | 요구사항, API, 개인정보 정책, 운영 정책 | 개인정보 및 운영 정책 변경에 해당 |
| `members.SUSPENDED` 로그인 허용 여부 확정 | 화면정의서, API 권한, 아키텍처 보안 | 회원 제재 정책 변경에 해당 |
| 신학 자문자 별도 계정/권한 모델 추가 여부 | 요구사항, ERD, API, 관리자 권한 | 역할/권한 추가는 의미 변경에 해당 |
| 한국어/영어 성경 본문 데이터의 최종 라이선스 및 저장 방식 확정 | 요구사항, ERD, 운영 정책 | 저작권 및 데이터 정책 변경에 해당 |
| API 명세 파일명과 내부 문서 버전 정합화 방식 결정 | 문서 운영 | 파일명 변경은 정리 변경이지만 외부 참조 영향 확인 필요 |

## 6. 다음 정합성 점검 기준

문서를 수정할 때는 다음 순서로 영향 범위를 확인한다.

1. 기능 ID, 우선순위, MVP 우선도, 수용 기준이 바뀌는가?
2. 역할/권한/데이터 보존/AI 검증 정책이 바뀌는가?
3. ERD 테이블, 상태값, enum이 요구사항과 충돌하는가?
4. 화면 상태와 API 응답이 같은 상태값을 사용하고 있는가?
5. 변경이 의미 변경이면 별도 변경안으로 분리했는가?
