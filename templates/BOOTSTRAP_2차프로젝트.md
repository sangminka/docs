> **이 문서의 목적** 5월 6일(주제 선정) ~ 6월 5일(시연 가정) 사이를 **하루 단위로 어떻게 운영할지** 결정해 둔 메타 플랜.
> **사용 방법** 주제 선정 직후 모두가 이 문서를 한 번 읽고, W0 킥오프 회의에서 합의된 항목만 그대로 따른다.

## 1. 가용 공수 (Capacity Budget)

```
1인 코어 가용:
  - 영업일 15일(W1~W3) × 6h = 90h
  - 회의/리뷰/디버깅/스위치 차감(약 -15h)
  - => 코어 개발 가용 ~ 75h/인

6인 팀 총 가용:
  - 540h (raw)
  - 460h (회의·리뷰 차감 후 실 작업 추정)

밸런스 권장 배분:
  - 기능 개발: 60% ≈ 275h
  - 테스트·QA: 15% ≈ 70h
  - DevOps·CI/CD: 10% ≈ 45h
  - 문서·시연 준비: 10% ≈ 45h
  - 버퍼: 5% ≈ 25h
```

> 75h/인이라는 제약을 잊으면, 모두가 "할 수 있을 것 같다"고 말한 뒤 W3 마지막 주에 무너집니다. **계획 단계에서 자르는 게 W3에 자르는 것보다 100배 쉽습니다.**

## 5. AI 에이전트 활용 가이드라인 (요약)

자세히는 [`templates/17_AI_에이전트_협업_가이드_template.md`](./templates/17_AI_에이전트_협업_가이드_template.md). 매일 지킬 것:

### ✅ 권장
- 단위 테스트 작성, 보일러플레이트, 데이터 변환, 정규식, SQL 초안
- 에러 메시지·문서·README 다듬기
- 새 라이브러리 학습용 예제 코드 (반드시 공식 문서로 검증)
- 코드 리뷰 1차 검토 (사람 리뷰 전 셀프 점검)

### ❌ 금지/위험
- 인증·결제·권한 로직을 통째로 맡기기
- 운영 DB에 SQL 실행 결정
- 시연 대본 단독 결정 (사람이 반드시 검수)
- 보안 의사결정 (해시 알고리즘, JWT 만료, 토큰 저장 위치 등)

### 🛡️ 모든 AI 산출물 6단계 게이트

```
PR 머지 전 통과해야 할 검증:
1. 빌드 성공 (CI)
2. 테스트 통과 (CI)
3. 린트/포맷 통과 (CI)
4. 프로젝트 컨벤션 일치 (사람 1차 확인)
5. 보안 점검 (시크릿·SQL·XSS·권한)
6. 사람 코드리뷰 + AGENTS.md 정합성 확인
```

### 📝 AI 활용 로그 (회고 정량 데이터)

PR 본문 또는 별도 노션 페이지에 기록:
- 어떤 도구로 어떤 작업을 도왔는가 (1줄)
- 환각/오류 발견 시 그 내용 (1줄)

W4 회고에서 위 로그를 집계하여 **"AI 사용 비율 / 환각 발견 횟수 / 재작업 횟수"**를 정량 회고로 발표.
