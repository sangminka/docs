## 6. 의존성 체인 (Critical Path)

```
W0 목-금: 주제 선정 → AGENTS.md → ERD v1 초안 → CI 1단계
   └─> W1 월: Entity·Repository (Lead 선행) — 팀 전체 blocked 제거
         └─> W1 화: ERD Lock + 공통 레이어(Security, 예외, DTO)
               └─> W1 수-목: 각 모듈 Service 시작 (병렬 4명)
                     └─> W1 금: CI/CD 완성 + 스테이징 배포
                           └─> W2: Controller + View + 외부 API
                                 └─> W3 화: Feature Freeze
                                       └─> W3 수: 통합 테스트
                                             └─> W3 목: 운영 배포
                                                   └─> W3 금: 시연 1차 리허설
```

**Critical Path 단축 전략 (3주 압축용)**

- **W0에 ERD 초안 확정** → W1 월부터 바로 Entity 작성 가능
- **W1 화요일까지** ERD Lock. 늦으면 전체 일정 깨짐.
- AI 에이전트로 Entity / Repository / 단위 테스트 골격 W1 후반 자동 생성 → 모듈별 Service 시작 1일 앞당김.
- **외부 API spike**: Lead가 W0-W1에 외부 API 1일 spike → W2 본 연동 막힘 최소화.
- **DevOps/QA가 W0-W1에 CI/CD 완성** → 모든 PR이 빠른 게이트 가짐.
