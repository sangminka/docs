### S-02. {{메인 유스케이스명}}

#### 화면 흐름

| 단계 | 화면 | 사용자 행동 | 시스템 반응 |
|:----:|------|-------------|--------------|
| 1 | `DB-01 대시보드` | "{{기능명}} 시작" 클릭 | `/feature/new` 이동 |
| 2 | `FE-01 입력 폼` | 필요 정보 입력 | 실시간 검증 |
| 3 | `FE-02 처리 중` | (대기) | 비동기 처리 + 진행률 표시 |
| 4 | `FE-03 결과` | 결과 확인, 저장/공유 | DB 저장 + 알림 |

#### 핵심 인터랙션 디테일
- **로딩 시간 > 1초**일 때만 스피너 표시 (1초 이하면 깜빡임 방지)
- **결과 화면**에 "다시하기" / "저장하기" / "공유하기" 3개 액션 노출
- 저장 시 토스트 + 대시보드의 "최근 활동"에 즉시 반영

## 3. 화면 ID 명명 규칙

```
{영역}-{번호}
LP : Landing Page
AU : Auth (로그인/가입/비밀번호 재설정)
OB : Onboarding
DB : Dashboard
FE : Feature (메인 기능별 접두사를 따로 만들어도 좋음 - 예: BK = Booking)
AD : Admin
ER : Error/Exception
```

- 화면 ID는 **06_화면_기능_정의서**, **04_API_명세서**의 화면 매핑과 동일하게 유지한다.
- 와이어프레임 파일명도 화면 ID로 통일: `LP-01_landing.png`
