# 요구사항 정의서 — QT-AI (큐티 AI 앱)

본 문서는 QT-AI 앱의 기준 요구사항 정의서다. 서비스의 핵심 방향, 사용자 흐름, 기능 범위, 화면 요구사항, 비기능 요구사항을 정리한다.

---

## 1. 문서 개요

### 1.1 한 줄 정의

QT와 성경을 잘 모르는 사용자도 오늘의 QT 본문을 쉽게 읽고, 한/영 성경과 AI가 미리 만들어 둔 해설·장면 시뮬레이터를 함께 보며, 모르는 사실은 AI에게 짧게 물어 확인하고, 개인 묵상 기록을 남길 수 있도록 돕는 심플한 QT 보조 앱.

### 1.2 문서의 목적

본 문서는 무엇을 만들 것인가와 그 기능을 만들기 위해 어떤 준비가 필요한가에 초점을 둔다. 상세 속성, 컬럼, 인덱스, API 스키마 등 물리 설계 영역은 별도 문서에서 다룬다.

이 문서의 목적은 다음 두 가지다.

1. 비즈니스 이해관계자가 서비스 전반을 한눈에 파악할 수 있게 한다.
2. 후속 단계인 테이블 설계, 기술 스택 선정, API 명세 작성의 입력 자료로 충분한 정보를 제공한다.

### 1.3 핵심 정책 요약

| 분류 | 정책 및 내용 | 관련 기능 |
| --- | --- | --- |
| 로그인 정책 | 앱 설치 후 회원가입·로그인 강제, 비회원 기능 전체 삭제 | F-01, F-04, F-10 |

### 1.4 미확정 사항

- AI 산출물의 교리·정합성 판정 주체: 이승욱 → 목사님 자문 → 강사님 협의
- 회원 탈퇴 데이터 법적 보존 기간 정확 일수: 강상민 조사
- 자체 디자인 시스템: 현재 베블레시아와 유사하므로 후속 단계에서 독자 디자인 필요

## 2. 서비스 방향

### 2.1 해결하려는 문제

| 대상 사용자 | 현재 문제 | 앱의 해결 |
| --- | --- | --- |
| 신앙 입문자 | 어디서부터 읽어야 할지 막막 | 오늘의 QT → 해설 → 시뮬레이터 → 묵상 기록의 단계형 흐름 |
| 시간 부족 직장인 | 두꺼운 주석책 펼칠 시간 없음 | 요약·해설 선택 열람, TTS로 이동 중 청취 |
| 묵상 방법론 부재 신앙인 | 자기중심 해석 위험 | 객관적 큐티 가이드와 사전 작성된 묵상 질문 |
| 기존 성경 AI 앱 사용자 | 답변 출처 불명·교리 검증 부재 | AI 해설 사전 생성·다층 검증, 출처 표시, 가치 판단형 답변 차단 |
| 본문 장면 상상 어려운 사용자 | 텍스트만으로 상황을 떠올리기 어려움 | QT 본문 옆 시뮬레이터 제공 |
| 모르는 단어·시대상에 막히는 사용자 | 검색해도 출처 신뢰가 어려움 | 사실 기반 Q&A AI 제공 |
| 자유 기록을 원하는 사용자 | QT 외 성경 구절이나 개인 기도·회개·감사 기록은 별도 메모장 필요 | 성경 화면의 설교 노트와 하단바 노트 영역의 개인 노트 제공 |
| 꾸준히 묵상하고 싶은 사용자 | 묵상 이력이 한눈에 안 보임 | 묵상 달력, 통계, 미션 진행률 제공 |

### 2.2 서비스 원칙

- 초심자도 사용할 수 있을 만큼 쉬운 앱을 지향한다.
- 커뮤니티 중심이 아니라 개인 묵상 공간을 중심으로 설계한다.
- 공유는 사용자가 원할 때만 선택적으로 동작한다.
- AI는 사용자가 작성하는 노트에 개입하지 않는다.
- AI는 사전 생성 콘텐츠와 사실 기반 단발 응답으로만 동작한다.
- 다중 턴 대화, 가치 판단, 신앙 평가, 단정적 설교형 답변은 제공하지 않는다.

### 2.3 핵심 사용 흐름

1. 사용자가 앱을 처음 실행한다.
2. 카카오 회원가입·로그인을 완료한다.
3. 첫 실행 튜토리얼에서 QT의 의미와 사용 순서를 확인한다.
4. 별도 홈 화면 없이 오늘의 QT 화면으로 진입한다.
5. 한/영 본문을 절 단위로 읽는다.
6. 오늘의 QT에서는 필요할 때 요약, 용어 풀이, 해설, 시뮬레이터, TTS를 사용한다.
7. 일반 성경 화면에서도 준비된 범위 안에서 해설, AI 질문 창, TTS를 사용한다.
8. 모르는 단어·시대상·역사는 우측 하단 AI 버튼으로 단발 질문한다.
9. 오늘의 QT 화면에서 묵상 노트를 작성하고, 일반 성경 화면에서는 절을 클릭해 설교 노트를 작성한다.
10. 하단바의 노트 영역에서 기도제목 노트, 회개 노트, 감사일기를 작성한다.
11. 원하면 묵상 기록을 닉네임 노출 상태로 공유한다.
12. 마이페이지에서 묵상 달력, 통계, 미션 진행률, 알림, 내 찬양 목록을 확인한다.

## 3. 사용자 진입과 인증

### 3.1 첫 실행 튜토리얼·QT 가이드 (F-07)

첫 실행 시 회원가입·로그인 직후 "QT가 처음이신가요?" 안내 화면을 통해 QT의 의미와 앱 사용 순서를 초심자 언어로 설명한다. 튜토리얼 종료 후 바로 오늘 QT 화면으로 진입한다.

**필요한 준비**

- 튜토리얼 콘텐츠 작성
- QT, 해설, 묵상, 적용, 기도 제목 등 용어 풀이 사전 작성
- MVP에서는 정적 리소스로 시작하고, 관리자 편집은 후속 검토

### 3.2 회원 인증 (F-04)

카카오 로그인으로 가입·로그인한다. 앱 설치 후 첫 진입 시 회원가입·로그인이 강제되며, 로그인 없이는 어떤 기능도 사용할 수 없다. 한 번 로그인하면 자동 로그인이 유지된다.

| 항목 | 정책 |
| --- | --- |
| 1차 공급자 | 카카오 로그인 |
| 로그인 강제 | 앱 설치 시 강제, 비회원 접근 없음 |
| 자동 로그인 | 활성화 |
| 회원 탈퇴 | MVP 포함 |
| 탈퇴 후 데이터 보존 | 법적 의무 기간, 정확 일수 후속 확정 |
| 추가 공급자 | 구글·애플 등 후속 검토 |

**필요한 준비**

- 카카오 로그인 연동 계약 및 자격증명 관리
- 인증 토큰 발급·검증·갱신 정책
- 자동 로그인 대응
- 강제 로그아웃 및 토큰 무효화 운영 방식
- 사용자 역할 구분: 일반, 관리자, 시스템 배치
- 회원 탈퇴 워크플로: 탈퇴 신청 → 본인 확인 → 즉시 비활성화 → 법적 보존 기간 경과 후 영구 삭제
- 탈퇴 후 보존 데이터 접근 권한 통제

## 4. 오늘의 QT 읽기 경험

### 4.1 성경·QT 본문 화면 (F-01)

오늘의 QT 본문을 절 단위로 한글·영어 비교해 읽는 화면이다. 앱 진입 시 별도 홈 화면 없이 이 화면으로 바로 들어온다. 모든 화면은 로그인된 사용자만 접근할 수 있다.

일반 성경 보기에서는 책·장·절을 자유롭게 조회할 수 있다. 일반 성경 화면에도 준비된 범위에서 해설, AI 질문 창, TTS를 제공한다. 단, 시뮬레이터와 오늘 QT 묵상 기록은 오늘 QT 본문 중심으로 제공한다.

**필요한 준비**

- 한국어·영어 성경 본문 데이터셋 확보
- 저작권 문제 없는 번역본 사용
- 날짜별 오늘의 QT 본문 범위 데이터
- 절 단위 본문 식별·정렬 좌표 체계
- 본문 갱신 시각 및 운영 정책
- 일반 성경 화면의 준비된 해설·AI 질문 창·TTS 제공 범위 기준

### 4.2 본문 요약·해설·용어 풀이 (F-08)

사용자는 오늘의 QT 본문 또는 해설이 준비된 일반 성경 본문에서 한 줄 요약, 어려운 단어 풀이, 절별 해설을 원할 때만 펼쳐서 본다. 강제 노출하지 않으며, 모든 항목 하단에 출처를 표시한다. 해설이 아직 준비되지 않았거나 검증을 통과하지 못한 절은 "준비된 해설 없음"으로 표시한다.

| 요소 | 위치 및 정책 |
| --- | --- |
| 해설 영역 | 해당 큰 대목 바로 밑 |
| 용어 풀이 | 해설 바로 밑, 단어 옆 별표 표시 |
| 별표 표시 조건 | 주석에 원래 단어 뜻이 있는 경우만 표시 |
| AI 질문 버튼 | 우측 하단, 본문을 가리지 않는 크기 |
| AI 질문 창 | 버튼 클릭 시 우측에서 슬라이드 인 |

**필요한 준비**

- 사전 생성 콘텐츠 조회 경로
- "준비된 해설 없음" 상태 표기 규칙
- 출처 표기 규칙
- 별표 표기의 일관된 디자인 토큰

### 4.3 시뮬레이터 보기 (F-12)

오늘의 QT 본문 중 일부 장면을 짧은 애니메이션으로 보여준다. 시뮬레이터는 QT 전용 기능이며, 일반 성경 자유 조회나 사용자 선택 성경 본문이 연결된 노트에는 적용하지 않는다.

**적용 범위**

- 오늘의 QT 본문에 한해 제공
- MVP에서는 일부 본문 우선 적용
- 장면 스크립트와 컴포넌트 라이브러리 조합으로 생성
- AI 자유 형식 HTML 생성은 제외

**필요한 준비**

- 장면 스크립트 스키마
- 사용할 수 있는 캐릭터·배경·소품·동작 컴포넌트 카탈로그
- QT 본문 좌표와 시뮬레이터 클립 매핑
- 검증 실패 시 미노출 처리

### 4.4 사실 기반 Q&A AI (F-15)

사용자가 본문을 읽다가 모르는 단어, 시대상, 역사적 사실을 만났을 때 우측 하단 AI 버튼을 눌러 짧게 질문한다. AI는 사실 기반으로만 단발 응답한다.

| 할 것 | 하지 않을 것 |
| --- | --- |
| 역사적 사실 답변 | 가치 판단 |
| 시대상·배경 설명 | 주관적 조언·권유 |
| 단어 뜻 설명 | 단정적 설교 |
| 주석·해설 기반 보조 설명 | 다중 턴 대화 |

**정책**

- 다중 턴 대화 미지원
- 앞선 질문 컨텍스트 유지 안 함
- 추상적·주관적 질문 차단
- 응답 하단 출처 표시
- F-14의 검증 기준 적용
- 일반 성경 화면과 QT 화면 모두 적용
- 로그인 필수

**필요한 준비**

- 외부 AI 모델 연동
- 사실 한정 응답 시스템 프롬프트 및 응답 스키마
- 가치 판단형 질문 차단 가드레일
- 응답 길이 제한과 말투 가이드라인
- 응답 캐싱 정책
- 사용자별 사용량 제한
- 응답 신고·피드백 경로

### 4.5 TTS (F-17)

본문과 해설을 음성으로 재생한다.

| 콘텐츠 | 적용 여부 |
| --- | --- |
| QT 본문 한/영 | 적용 |
| 일반 성경 본문 | 적용 |
| 해설 | 적용 |
| 묵상 노트 | 미적용 |
| 닉네임 나눔 게시글 | 미적용 |

**필요한 준비**

- TTS 엔진 선택
- 음성·속도·언어 선택 UI
- 재생, 일시정지, 이전 절, 다음 절 컨트롤
- 백그라운드 재생 정책
- 잠금화면·블루투스 미디어 컨트롤 후속 검토

## 5. 묵상 작성과 개인 기록

### 5.1 노트 작성·관리 (F-03)

사용자는 노트 종류에 따라 서로 다른 화면에서 노트를 작성한다. 묵상 노트는 오늘의 QT 화면에서 작성하고, 설교 노트는 일반 성경 화면에서 절을 클릭해 작성한다. 기도제목 노트, 회개 노트, 감사일기는 하단바의 "노트" 영역에서 작성한다. 노트는 5개 카테고리로 나뉘며, 묵상 노트만 4섹션 구조를 사용하고 나머지 노트는 메모장처럼 1개 입력 영역에 자유롭게 작성한다.

| 카테고리 | 작성 진입점 | 입력 구조 | 작성 방식 |
| --- | --- | --- | --- |
| 묵상 노트 | 오늘의 QT 화면 | 4섹션 | 느낀 점, 기억할 구절, 적용할 점, 기도 |
| 설교 노트 | 일반 성경 화면에서 절 클릭 | 1섹션 | 선택한 구절을 보며 설교 메모 |
| 기도제목 노트 | 하단바 노트 영역 | 1섹션 | 기도제목을 자유롭게 메모 |
| 회개 노트 | 하단바 노트 영역 | 1섹션 | 회개 내용을 자유롭게 메모 |
| 감사일기 | 하단바 노트 영역 | 1섹션 | 감사 내용을 자유롭게 메모 |

묵상 노트의 4섹션은 다음과 같다.

| 섹션 |
| --- |
| 느낀 점 |
| 기억할 구절 |
| 적용할 점 |
| 기도 |

묵상 노트는 일부 섹션만 채워도 저장 가능하다. 설교 노트, 기도제목 노트, 회개 노트, 감사일기는 1섹션 본문만 작성해도 저장 가능하다. AI는 사용자가 작성하는 모든 노트에 개입하지 않는다.

### 5.2 저장 방식

| 동작 | 트리거 | UI |
| --- | --- | --- |
| 임시 저장 | 사용자가 "임시저장" 버튼 클릭 | "임시저장됨" 표시 |
| 저장 | 사용자가 "저장" 버튼 클릭 | "저장 완료되었습니다" 표시 |

자동 저장은 어느 단계에서도 일어나지 않는다. 모든 노트 작성 화면에는 임시저장과 저장 버튼 2개가 있으며, 둘 다 사용자가 직접 클릭해야 동작한다.

**저장 방식 원칙**

- 사용자가 의도하지 않은 상태가 저장되는 경험을 방지한다.
- 백스페이스 등 편집 중 행동이 기존 내용을 덮어쓰는 상황을 막는다.

**필요한 준비**

- 임시저장 버튼·저장 버튼 UI 노출 정책
- 임시본과 확정본 데이터 모델 구분
- 사용자별 일자별 QT 묵상 노트 1건 중복 방지
- 설교 노트, 기도제목 노트, 회개 노트, 감사일기는 일자별 중복 제한 없음
- 카테고리별 입력 구조 구분: 묵상 노트 4섹션, 그 외 노트 1섹션
- 노트별 작성 진입점 구분: QT 화면 / 일반 성경 화면 절 클릭 / 하단바 노트 영역
- 묵상 달력 연동
- 공개·비공개 토글 정책
- AI와 노트 작성 영역의 권한 분리

### 5.3 노트 카테고리와 사용자 선택 성경 본문 (F-16)

노트는 5개 카테고리로 구분되지만, 작성 진입점은 카테고리별로 다르다. 사용자 선택 성경 본문은 일반 자유 텍스트가 아니라, 성경 책·장·절에서 사용자가 직접 고른 구절을 의미한다. 설교 노트는 일반 성경 화면에서 절을 클릭해 작성하며, 기도제목 노트·회개 노트·감사일기는 성경 본문 연결 없이 하단바의 "노트" 영역에서 메모장처럼 작성한다.

| 카테고리 | 그룹 | 작성 진입점 | 성경 본문 연결 | 사용 케이스 |
| --- | --- | --- | --- | --- |
| 묵상 노트 | QT 그룹 | 오늘의 QT 화면 | 오늘 QT 본문 자동 연결 | 그날의 QT 묵상 |
| 설교 노트 | 성경 그룹 | 일반 성경 화면에서 절 클릭 | 클릭한 절 또는 선택한 여러 절 연결 | 설교를 들으며 해당 구절 메모 |
| 기도제목 노트 | 개인 그룹 | 하단바 노트 영역 | 본문 좌표 무관 | 개인·교회·이웃 기도제목 정리 |
| 회개 노트 | 개인 그룹 | 하단바 노트 영역 | 본문 좌표 무관 | 개인 회개 기록 |
| 감사일기 | 개인 그룹 | 하단바 노트 영역 | 본문 좌표 무관 | 일상 감사 기록 |

**사용자 인지 구조**

| 그룹 | 포함 카테고리 |
| --- | --- |
| QT 그룹 | 묵상 노트 |
| 성경 그룹 | 설교 노트 |
| 개인 그룹 | 기도제목 노트, 회개 노트, 감사일기 |

**정책**

- 묵상 노트는 오늘의 QT 화면에서 작성
- 설교 노트는 일반 성경 화면에서 절을 클릭해 작성
- 기도제목 노트, 회개 노트, 감사일기는 하단바의 "노트" 영역에서 작성
- 묵상 노트만 4섹션 구조 사용
- 설교 노트, 기도제목 노트, 회개 노트, 감사일기는 1섹션 메모장형 구조 사용
- 제목 입력 가능
- 설교 노트는 한 절 또는 여러 절 선택 가능
- 기도제목 노트, 회개 노트, 감사일기는 성경 구절 선택 없이 작성 가능
- 공개 범위는 비공개 또는 닉네임 노출 공유
- 시뮬레이터는 적용하지 않음
- 설교 노트 등 성경 본문이 연결된 노트는 해당 본문에 준비된 해설이 있을 때만 해설 노출
- 준비된 해설이 없으면 "준비된 해설 없음" 표시

**필요한 준비**

- 오늘의 QT 화면 내 묵상 노트 작성 진입점
- 일반 성경 화면에서 절 클릭 후 설교 노트 작성으로 이어지는 UI
- 하단바 "노트" 영역의 기도제목 노트·회개 노트·감사일기 작성 UI
- 제목 입력 필드 및 글자 수 정책
- 카테고리 enum
- 카테고리별 입력 구조: 4섹션 / 1섹션
- 공개·비공개 토글
- 마이페이지 목록·검색에서 카테고리 구분
- 설교 노트 중복 제한 정책
- 설교 노트의 사용자 선택 성경 본문 좌표와 사전 생성 해설 범위 매칭 로직

## 6. 찬양과 개인 라이브러리

### 6.1 찬양 큐레이션·내 찬양 목록 (F-09)

운영자가 사전 등록한 찬양 곡 리스트를 사용자가 둘러보고, 마음에 드는 곡을 본인 찬양 목록에 저장한다. AI는 찬양을 추천하지 않는다.

| 소스 | 설명 |
| --- | --- |
| 기본 찬양곡 | 저작권 없는 무료곡 몇 곡 자체 보유 |
| 사용자 음원 | 디바이스에 있는 찬양곡 불러오기 가능 |
| 라이브러리 | 사용자가 저장한 곡 목록 |

서버에는 곡 메타데이터만 저장한다. 가사와 음원 파일은 서버에 저장하지 않는다.

**필요한 준비**

- 운영자 큐레이션 등록 화면
- 자체 보유곡 저작권 안전성 검증
- 사용자 디바이스 음원 접근 권한 요청 UX
- 사용자 라이브러리 저장·해제 방식

## 7. 공유와 커뮤니티

### 7.1 닉네임 나눔 공간 (F-10)

사용자가 명시적으로 선택한 묵상 기록만 닉네임이 노출된 상태로 공유된다. 인스타그램식 피드가 아니라 블로그식 조용한 나눔에 가깝다. 좋아요, 댓글, 신고의 최소 반응만 제공한다.

**공유 정책**

- 기본값은 비공개
- 사용자가 공유를 선택한 기록만 노출
- 공유 시 닉네임이 자동 노출
- 별도 동의 절차 없음
- 가입 시 닉네임 지정이 노출 전제 동의로 간주
- 익명 게시 없음
- 닉네임 중복 방지 필요

**반응 정책**

- 좋아요 제공
- 댓글은 작성자가 ON/OFF 선택
- 신고 제공
- 팔로우, 랭킹, 실시간 댓글은 MVP 제외

**필요한 준비**

- 공유 토글 UI
- 닉네임 중복 방지
- 댓글 ON/OFF 정책
- 신고 접수 및 처리 흐름
- 공유 취소 또는 비공개 전환 처리
- 부적절 콘텐츠 운영 정책

### 7.2 인앱 알림 (F-05)

사용자가 인앱 알림 목록에서 자기 활동 결과에 대한 로그형 알림을 확인한다. MVP는 실시간 푸시 알림을 제공하지 않는다. 마이페이지의 알림 위젯을 열었을 때 가용한 알림을 보여주는 수준까지 보장한다.

**알림 유형**

- 내 닉네임 나눔 글에 좋아요가 달림
- 내 닉네임 나눔 글에 댓글이 달림
- 내가 한 신고가 처리됨
- 시스템 공지 발행

**필요한 준비**

- 알림 유형 분류 체계
- 알림 생성 이벤트 트리거
- 읽음·미읽음 상태 관리
- 마이페이지 알림 위젯 UI

## 9. AI 콘텐츠 생성과 검증

### 9.1 AI 콘텐츠 사전 생성 (F-02)

사용자에게 노출할 절별 해설과 시뮬레이터 클립을 정해진 시각에 배치로 미리 생성한다. 사용자 요청 경로에서는 사전 생성 콘텐츠 조회만 일어난다. 사실 기반 Q&A AI는 별도 경로로 동작한다.

해설은 본문만 보고 자유 생성하지 않는다. 시중에 존재하는 성경 주석의 영어 원문을 입력으로 받아 재해석·번역한다. 검증을 통과한 결과만 사용자에게 노출된다.

**필요한 준비**

- 외부 AI 모델 연동 계약 및 자격증명 관리
- 입력용 영어 주석 데이터 확보
- 검증 비교용 한국어 주석 데이터 확보
- 본문 좌표 기반 주석 조회 방식
- 배치 스케줄러와 관리자 수동 트리거
- 생성 결과 상태 관리: 검토 대기, 노출, 반려
- 출처 표기 정책
- AI 산출물 말투 가이드라인

### 9.2 AI 산출물 품질 관리 (F-14)

사용자에게 노출되는 해설, 시뮬레이터 스크립트, 사실 기반 Q&A 응답은 검증 절차를 거친다. AI 산출물은 정통 기준에서 벗어나지 않도록 검증하고, 사용자에게는 검증을 통과한 결과만 노출한다.

**다층 검증 구조**

| 층 | 검증 대상 | 검증 주체 | 통과 못 한 경우 |
| --- | --- | --- | --- |
| 1층 | 스키마, 본문 좌표, 금지 키워드, 원문 직접 복제, 자산 호출 규칙 | 자동 | 즉시 반려 |
| 2층 | 사실 정합성, 검증 체크리스트, 말투, 초심자 이해 가능성 | 자동 | 반려 또는 검토 대기 |
| 3층 | 어조, 뉘앙스, 문화적 적절성, 신학 자문 의견 | 운영진·신학 자문 | 기준에 재반영 |

**필요한 준비**

- 신학 기준 문서 확정
- 검증 체크리스트 항목화
- 금지 표현·금지 주제 목록 운영 정책
- AI 생성 지시 자산 버전 관리
- 평가 셋 운영
- 검증 결과 로깅
- 사용자 신고 사례를 평가 셋에 환류시키는 흐름

### 9.3 관리자 운영 기능 (F-06)

운영자는 콘텐츠와 사용자 활동을 관리한다.

**관리 대상**

- 오늘의 QT 본문 등록·수정
- 사용자 노출 해설 등록·수정
- AI 산출물 검증·반려 로그 조회
- 닉네임 나눔 신고 처리
- 찬양 큐레이션 관리
- 특정 본문 재생성 수동 트리거
- 운영 지표 조회
- 공지사항 발행

**감사 로그 필수 대상**

- 오늘 QT 본문 변경
- 사용자 노출 해설 변경
- AI 재생성 트리거
- 찬양 큐레이션 변경
- 닉네임 나눔 삭제·비공개 처리
- 사용자 제재·권한 변경

**저작권 보호 정책**

검증용 한국어 주석은 관리자 화면에서도 노출하지 않는다. 자체 제작자만 접근 가능한 별도 경로를 유지한다.

## 10. 화면 요구사항

총 7개 영역으로 구성한다. 모든 화면은 로그인 후에만 접근 가능하다.

| # | 영역 |
| --- | --- |
| 1 | 온보딩·튜토리얼·회원가입/로그인 |
| 2 | 오늘의 QT: 성경 본문, 해설, 용어 풀이, 시뮬레이터, AI 질문 창, TTS |
| 3 | 일반 성경 자유 조회: 한/영 비교, 준비된 해설, AI 질문 창, TTS |
| 4 | 노트: 기도제목 노트, 회개 노트, 감사일기 작성 및 노트 목록 |
| 5 | 마이페이지: 묵상 달력, 통계, 미션 진행률, 알림, 내 찬양 목록 |
| 6 | 닉네임 나눔: 피드, 상세, 신고 |
| 7 | 관리자 콘솔 |

각 화면별 상세 와이어프레임은 별도 UI 문서에서 다룬다. 디자인 시스템은 후속 단계에서 베블레시아와 차별화된 자체 디자인으로 정리한다.

## 11. 범위 정의

### 11.1 MVP 포함

- 시연 주간 서비스 가용성 99%
- 시연 시나리오 성공률 100%
- 사전 리허설 2회 통과 기준
- 장애 시 로컬 재기동 및 백업 영상으로 폴백

- 사용자 노출 해설은 검증 통과 결과만 게시

## 15. 용어 정의

### 15.1 용어 통일

| 기존 용어 | 확정 용어 | 의미 |
| --- | --- | --- |
| 본문 / 공문 | 성경 | 성경 책 전체 |
| 오늘의 본문 | QT | 하루치 QT 본문 |
| 자유 본문 | 사용자 선택 성경 본문 | 사용자가 성경 책·장·절에서 직접 고른 구절 |
| 사용자 노트 | 노트 | 앱에서 작성하는 기록. 카테고리별로 작성 진입점이 다름 |
| 주석 | 해설 | AI 생성 + 주석 기반 설명 |
| 이단성 | 검증 | AI 생성 해설의 정합성 판정 |

### 15.2 용어 사전

| 용어 | 정의 |
| --- | --- |
| QT | 개인이 매일 일정 시간 성경을 읽고 묵상·적용·기도를 정리하는 신앙 활동 |
| 오늘의 QT | 매일 정해지는 그날의 묵상용 성경 본문 |
| 사용자 선택 성경 본문 | 사용자가 성경 책·장·절에서 직접 고른 구절. 일반 자유 텍스트 입력이 아니다 |
| 노트 | 사용자가 앱에서 작성하는 기록 묶음. 묵상 노트, 설교 노트, 기도제목 노트, 회개 노트, 감사일기로 구분된다 |
| 묵상 노트 | 오늘의 QT 화면에서 작성하며 오늘의 QT 본문에 연결되는 4섹션 QT 노트 |
| 설교 노트 | 일반 성경 화면에서 절을 클릭해 작성하며 사용자 선택 성경 본문에 연결되는 1섹션 메모장형 노트 |
| 기도제목 노트 | 하단바 노트 영역에서 작성하며 본문 좌표 없이 1섹션으로 작성하는 기도제목 정리용 노트 |
| 회개 노트 | 하단바 노트 영역에서 작성하며 본문 좌표 없이 1섹션으로 작성하는 개인 회개 기록 노트 |
| 감사일기 | 하단바 노트 영역에서 작성하며 본문 좌표 없이 1섹션으로 작성하는 일상 감사 기록 노트 |
| 해설 | 본문 이해를 돕는 AI 생성 + 주석 기반 설명 |
| 시뮬레이터 | 오늘 QT 본문의 장면을 보여주는 짧은 애니메이션 |
| 장면 스크립트 | 시뮬레이터용 구조화 데이터 |
| 컴포넌트 라이브러리 | 시뮬레이터 장면 구성 자산 카탈로그 |
| 찬양 큐레이션 | 운영자가 사전에 등록·관리하는 찬양 목록 |
| 닉네임 나눔 | 사용자가 묵상 기록을 닉네임 노출 상태로 공유하는 공간 |
| 사실 기반 Q&A AI | 단어·시대상·역사 한정으로 단발 응답하는 AI |
| 다층 검증 | AI 산출물 노출 전 형식·의미·인간 검토를 거치는 품질 검사 |
| 검증 체크리스트 | 산출물이 신학 기준에서 벗어나는지 점검하는 항목 묶음 |
| 신학 기준 문서 | 본 서비스가 정통으로 인정하는 신앙 기준 출처 묶음 |
